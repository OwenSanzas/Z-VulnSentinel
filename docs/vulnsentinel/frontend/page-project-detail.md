## 区域说明

| 区域 | 说明 |
|------|------|
| **项目信息头** | 项目名、仓库 URL、联系人（被通知人）、最新更新时间、当前版本（tag） |
| **Client Vulnerabilities** | 该项目的所有 client_vulns，含统计数字（recorded / reported / confirmed / fixed） |
| **Dependencies** | 该项目使用的所有依赖库，含版本约束、实际版本、来源、当前漏洞状态 |
| **Snapshots** | 我们存储的客户代码快照，记录何时获取、哪个版本、当前活跃的是哪个 |

## Snapshot 策略

| 触发条件 | 说明 |
|----------|------|
| **tag push** | 客户发布新版本时自动拉取快照 |
| **manual** | 团队手动触发快照 |
| **scheduled** | 定期拉取（可配置频率） |

每个 snapshot 包含：完整源码、提取的依赖列表、构建产物（bitcode / 调用图）。新 snapshot 生成后自动成为 active，旧的归档保留。
